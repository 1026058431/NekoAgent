# LangChain 哲学

LangChain 由几个核心信念驱动：

## 核心信念

1. **大型语言模型 (LLMs) 是伟大、强大的新技术**
2. **当您将 LLMs 与外部数据源结合时，它们会变得更好**
3. **LLMs 将改变未来应用程序的外观**
4. **具体来说，未来的应用程序将越来越具有智能性**
5. **这种转型仍处于非常早期的阶段**
6. **虽然构建这些智能应用程序的原型很容易，但构建足够可靠以投入生产的智能体仍然非常困难**

## 核心重点

通过 LangChain，我们有两个核心重点：

### 1. 使开发人员能够使用最佳模型

不同提供商暴露不同的 API，具有不同的模型参数和不同的消息格式。标准化这些模型输入和输出是一个核心重点，使开发人员能够轻松切换到最新的最先进模型，避免锁定。

### 2. 使模型易于编排更复杂的流程

模型应该用于不仅仅是文本生成 - 它们还应该用于编排与其他数据交互的更复杂流程。LangChain 使定义 LLMs 可以动态使用的工具变得容易，并帮助解析和访问非结构化数据。

## 历史发展

### 2022-10-24 v0.0.1
- 在 ChatGPT 发布前一个月，LangChain 作为 Python 包发布
- 包含两个主要组件：LLM 抽象和链（用于常见用例的预定计算步骤）
- 名称 LangChain 来自 Language（如语言模型）和 Chains

### 2022-12
- 第一个通用智能体被添加到 LangChain
- 这些通用智能体基于 ReAct 论文（ReAct 代表推理和行动）
- 使用 LLMs 生成表示工具调用的 JSON，然后解析该 JSON 以确定要调用的工具

### 2023-01
- OpenAI 发布 Chat Completion API
- 模型从接收字符串并返回字符串演变为接收消息列表并返回消息
- LangChain 更新以使用消息列表

### 2023-01
- LangChain 发布 JavaScript 版本
- LLMs 和智能体将改变应用程序的构建方式，JavaScript 是应用程序开发人员的语言

### 2023-02
- LangChain Inc. 围绕开源 LangChain 项目成立为公司
- 主要目标是使智能智能体无处不在

### 2023-03
- OpenAI 在其 API 中发布函数调用
- 允许 API 显式生成表示工具调用的有效载荷
- LangChain 更新以使用此作为工具调用的首选方法

### 2023-06
- LangSmith 作为闭源平台由 LangChain Inc. 发布，提供可观察性和评估
- 构建智能体的主要问题是使它们可靠，LangSmith 就是为解决这一需求而构建的
- LangChain 更新以与 LangSmith 无缝集成

### 2024-01 v0.1.0
- LangChain 发布 0.1.0，这是第一个非 0.0.x 版本
- 行业从原型发展到生产，因此 LangChain 增加了对稳定性的关注

### 2024-02
- LangGraph 作为开源库发布
- 原始 LangChain 有两个重点：LLM 抽象和用于开始常见应用程序的高级接口；但它缺少允许开发人员控制其智能体确切流程的低级编排层
- 在构建 LangGraph 时，我们从构建 LangChain 的经验中学习，并添加了我们发现需要的功能：流式传输、持久执行、短期记忆、人工干预等

### 2024-06
- LangChain 拥有超过 700 个集成
- 集成从核心 LangChain 包中分离出来，要么移动到它们自己的独立包中（对于核心集成），要么移动到 langchain-community

### 2024-10
- LangGraph 成为构建任何超过单个 LLM 调用的 AI 应用程序的首选方式
- 大多数链和智能体在 LangChain 中被标记为已弃用，并提供了如何将它们迁移到 LangGraph 的指南
- 在 LangGraph 中仍然创建了一个高级抽象：智能体抽象

### 2025-04
- 模型 API 变得更加多模态
- 模型开始接受文件、图像、视频等
- 我们相应地更新了 langchain-core 消息格式，以允许开发人员以标准方式指定这些多模态输入

### 2025-10-20 v1.0.0
- LangChain 发布 1.0，有两个主要变化：
  - 完全重构 langchain 中的所有链和智能体
  - 所有链和智能体现在都被替换为只有一个高级抽象：基于 LangGraph 构建的智能体抽象
  - 标准消息内容格式：模型 API 从返回具有简单内容字符串的消息演变为更复杂的输出类型 - 推理块、引用、服务器端工具调用等
  - LangChain 演进其消息格式以跨提供商标准化这些格式
